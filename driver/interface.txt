CHARACTER DEVICES

--------------------------------------------------------------------------------
Device 0: VGA (Video Graphics Array)
--------------------------------------------------------------------------------
Type: Character Device
File: driver/char/vga.c
Registration: vga_init()

Description:
  Direct VGA text-mode output driver providing 80x25 character display.
  Hardware-level VGA operations including cursor control and scrolling.

Operations:
  read(scnd_id)
    - NOT SUPPORTED (always returns 0)
    - VGA hardware does not provide read capability
  
  write(scnd_id, char)
    - Writes a character to the VGA text buffer
    - Parameters:
        scnd_id: Unused (ignored)
        char: ASCII character to display
    - Special characters:
        '\n' - Newline (move to start of next line)
        '\r' - Carriage return (move to start of current line)
        '\b' - Backspace (move cursor back and clear character)
    - Automatically handles screen scrolling when reaching bottom
    - Updates hardware cursor position
    - Return: 0 on success
  
  ioctl(prim_id, scnd_id, command)
    - NOT IMPLEMENTED (returns -1)

Notes:
  - Direct hardware access to VGA buffer at 0xC00B8000
  - 80 columns × 25 rows text mode
  - Default color: light grey on black
  - Automatic scrolling when screen fills
  - Hardcoded address (standard VGA legacy location)

--------------------------------------------------------------------------------
Device 1: PIT (Programmable Interval Timer)
--------------------------------------------------------------------------------
Type: Character Device  
File: driver/char/pit.c
Registration: pit_init()

Description:
  8253/8254 PIT driver providing system timing and tick counting.
  Configured to generate interrupts at specified frequency (typically 100Hz).
  Uses unified interrupt manager for both PIC and APIC modes.

Operations:
  read(scnd_id)
    - Returns LOW BYTE of the current tick counter
    - Parameters:
        scnd_id: Unused (ignored)
    - Return: (tick_count & 0xFF) as char
    - Note: Only returns lowest 8 bits, use pit_get_ticks() for full value
  
  write(scnd_id, char)
    - NOT SUPPORTED (no operation)
    - Parameters ignored
    - Return: 0 (success, but does nothing)
  
  ioctl(prim_id, scnd_id, command)
    - NOT IMPLEMENTED (returns NULL)

Notes:
  - Generates IRQ0 interrupts at configured frequency
  - Internal tick counter incremented on each interrupt
  - For full 32-bit tick count, use pit_get_ticks() kernel function
  - Default frequency: 100Hz (configurable via pit_init())
  - Uses interrupt_eoi() and interrupt_enable_irq() for mode independence
  - Registered as /dev/pit0 in devfs

--------------------------------------------------------------------------------
Device 2: TTY (Terminal Emulator)
--------------------------------------------------------------------------------
Type: Character Device
File: driver/char/tty.c  
Registration: tty_init()

Description:
  ANSI terminal emulator built on top of VGA. Provides escape sequence
  parsing, cursor control, color support, and advanced text formatting.
  Input is provided via keyboard driver (device 3).

Operations:
  read(scnd_id)
    - Reads a character from the keyboard
    - Parameters:
        scnd_id: Unused (ignored)
    - Returns: ASCII character from keyboard buffer, or 0 if buffer empty
    - Implementation: Forwards to keyboard driver via cread(3, 0)
    - Note: Non-blocking - returns immediately even if no key pressed
  
  write(scnd_id, char)
    - Writes a character to TTY with ANSI escape sequence processing
    - Parameters:
        scnd_id: Unused (ignored)
        char: Character to output (may be part of escape sequence)
    - Control characters:
        '\n' - Line feed (new line)
        '\r' - Carriage return
        '\b' - Backspace with erase
        '\t' - Tab (8-space alignment)
        '\a' - Bell (currently ignored)
        '\033' (ESC) - Start of escape sequence
    - ANSI Escape Sequences Supported:
        ESC[nA    - Cursor up n lines
        ESC[nB    - Cursor down n lines
        ESC[nC    - Cursor forward n columns
        ESC[nD    - Cursor backward n columns
        ESC[n;mH  - Move cursor to row n, column m
        ESC[2J    - Clear entire screen
        ESC[K     - Erase from cursor to end of line
        ESC[0m    - Reset all attributes
        ESC[1m    - Bold/bright
        ESC[30-37m - Foreground colors (ANSI colors)
        ESC[40-47m - Background colors (ANSI colors)
    - Return: 0 on success
  
  ioctl(prim_id, scnd_id, command)
    - Send control commands to TTY
    - Parameters:
        prim_id: Must be 2 (TTY device)
        scnd_id: Unused (ignored)
        command: Command code
    - Supported Commands:
        0x1 (TTY_IOCTL_CLEAR) - Clear screen and reset cursor to (0,0)
    - Return: 0 on success, -1 on unknown command
    - Example: ioctl(2, 0, 0x1) clears the screen

Notes:
  - Built on VGA hardware but provides ANSI compatibility
  - 80x25 character display
  - Full color support (16 foreground, 8 background)
  - Tab stops at every 8 characters
  - ANSI color mapping to VGA color palette
  - ioctl command 0x1 provides programmatic screen clearing
  - Registered as /dev/tty0 in devfs

--------------------------------------------------------------------------------
Device 3: Keyboard (PS/2 Keyboard)
--------------------------------------------------------------------------------
Type: Character Device
File: driver/char/kbd.c
Registration: kbd_init()

Description:
  PS/2 keyboard driver with IRQ1 interrupt handling. Converts scancodes to
  ASCII characters using translation tables. Supports modifier keys (Shift,
  Caps Lock) and buffers input in a 128-byte circular buffer. No automatic
  echo - applications must explicitly output received characters.
  Uses unified interrupt manager for both PIC and APIC modes.

Operations:
  read(scnd_id)
    - Reads next character from keyboard buffer
    - Parameters:
        scnd_id: MUST be 0 (enforced, returns 0 if not)
    - Returns: ASCII character from buffer, or 0 if buffer empty
    - Non-blocking: Returns immediately
    - Note: NO automatic echo to screen - silent input
  
  write(scnd_id, char)
    - NOT SUPPORTED (always returns -1)
    - Keyboard is input-only device
  
  ioctl(prim_id, scnd_id, command)
    - NOT SUPPORTED (always returns -1)
    - No control commands implemented

Scancode Translation:
  - Set 1 scancodes (PC/AT standard)
  - Make codes: 0x01-0x58 (key pressed)
  - Break codes: 0x81-0xD8 (key released, ignored)
  - Special Keys:
      Enter → '\n' (0x0A)
      Backspace → '\b' (0x08)
      Tab → '\t' (0x09)
      Escape → 0x1B
      Space → ' '
  - Modifier Support:
      Shift (Left/Right) - Uppercase and symbols
      Caps Lock - Toggle letter case
      Ctrl, Alt - Not yet implemented
  - Unsupported keys silently ignored (F1-F12, etc.)

Buffer Behavior:
  - Circular buffer: 128 bytes
  - On overflow: New characters dropped (buffer full)
  - Thread-safe: Modified only in IRQ1 handler

Notes:
  - IRQ1 (keyboard interrupt) on vector 33
  - Reads from port 0x60 (KBD_DATA_PORT, hardcoded standard)
  - No echo - application must output received chars
  - scnd_id validation enforced (must be 0)
  - Used by TTY for input via cread(3, 0)
  - 100% interrupt-driven, no polling
  - Uses interrupt_eoi() and interrupt_enable_irq() for mode independence
  - Registered as /dev/kbd0 in devfs

Device 9: RTC (Real-Time Clock)
--------------------------------------------------------------------------------
Type: Character Device
File: driver/char/rtc.c
Registration: rtc_init()

Description:
  CMOS/RTC driver providing system date and time from hardware real-time clock.
  Reads current time on initialization and provides read-only access.

Operations:
  read(scnd_id)
    - NOT SUPPORTED (returns 0)
    - Use rtc_get_datetime() kernel function instead
  
  write(scnd_id, char)
    - NOT SUPPORTED (returns -1)
    - RTC is read-only in this implementation
  
  ioctl(prim_id, scnd_id, command)
    - NOT SUPPORTED (returns -1)

API Functions:
  - rtc_get_datetime(rtc_datetime_t *dt) - Get current date/time

Notes:
  - Reads from CMOS ports 0x70 (address) and 0x71 (data)
  - Hardcoded ports (standard CMOS/RTC legacy location)
  - Provides year, month, day, hour, minute, second
  - BCD to binary conversion performed automatically
  - Time read during initialization
  - Registered as /dev/rtc0 in devfs

Devices 10-13: Serial Ports (serial0-serial3)
--------------------------------------------------------------------------------
Type: Character Device
File: driver/char/serial.c
Registration: serial_init()

Description:
  Multi-port serial UART driver with dynamic discovery. Automatically detects
  serial ports via PCI UART controllers and legacy ISA port probing. Supports
  up to 4 serial ports. serial0 is used for kernel logging (printk).
  
  **DYNAMIC DISCOVERY** - No hardcoded addresses! Uses:
  1. PCI bus scan for UART controllers (class 0x07, subclass 0x00)
  2. Legacy ISA port probing (COM1-COM4: 0x3F8, 0x2F8, 0x3E8, 0x2E8)
  
  All detected ports are initialized to 115200 baud, 8N1.

Device Mapping:
  10 = serial0 (first detected port, typically COM1) - **kernel logging**
  11 = serial1 (second detected port, typically COM2)
  12 = serial2 (third detected port, typically COM3)
  13 = serial3 (fourth detected port, typically COM4)

Operations:
  read(scnd_id)
    - Reads a byte from serial port if available
    - Parameters:
        scnd_id: Port ID (0-3 for serial0-serial3)
    - Returns: Character from port, or 0 if no data available
    - Non-blocking: Returns immediately
    - Checks Line Status Register for data ready
  
  write(scnd_id, char)
    - Writes a character to serial port
    - Parameters:
        scnd_id: Port ID (0-3 for serial0-serial3)
        char: Character to write
    - Automatic CRLF conversion: '\n' → '\r\n'
    - Waits for transmit buffer to be empty (with timeout)
    - Return: 0 on success, -1 on invalid port
  
  ioctl(prim_id, scnd_id, command)
    - NOT IMPLEMENTED (returns NULL)

Port Detection:
  - PCI Detection:
      • Scans PCI bus for class 0x07 (Simple Communications)
      • Subclass 0x00 (Serial Controller, 16550-compatible)
      • Reads BAR0 for I/O port address
      • Supports I/O port based UARTs
      • MMIO UARTs detected but not yet supported
  
  - ISA Detection:
      • Tests ports 0x3F8, 0x2F8, 0x3E8, 0x2E8 (COM1-COM4)
      • Validates by checking scratch register
      • Pattern test: write 0x55, read, write 0xAA, read
      • Only registers ports that respond correctly

Port Types:
  - SERIAL_TYPE_ISA:  Legacy ISA I/O port
  - SERIAL_TYPE_PCI:  PCI UART (I/O port based)
  - SERIAL_TYPE_MMIO: Memory-mapped I/O (detected but not supported yet)

Configuration:
  - Baud rate: 115200 (divisor = 1)
  - Data bits: 8
  - Parity: None
  - Stop bits: 1 (8N1)
  - FIFO: Enabled with 14-byte threshold
  - Flow control: DTR/RTS enabled

API Functions:
  - serial_write_char(char) - Write to serial0 (for printk)
  - serial_get_port_count() - Get number of detected ports
  - serial_get_port(port_id) - Get port descriptor

Notes:
  - **NO HARDCODED ADDRESSES** - fully dynamic
  - Automatically registers all detected ports in devfs
  - serial0 used by printk for kernel logging
  - Supports both PCI UART cards and legacy COM ports
  - Port existence verified before registration
  - Maximum 4 ports (SERIAL_MAX_PORTS)
  - Non-blocking I/O
  - No interrupt support (polling mode)
  - Registered as /dev/serial0-3 in devfs

Examples:
  - cwrite(10, 0, 'A') - Write 'A' to serial0
  - cread(11, 1, buf) - Read from serial1
  - Serial port at 0x3F8 becomes serial0 if detected first
  - PCI UART at custom address becomes serial1 if detected second

Devices 0-3: IDE Hard Disks (hda, hdb, hdc, hdd)
--------------------------------------------------------------------------------
Type: Block Device
File: driver/block/ide.c
Registration: ide_init()

Description:
  IDE/ATA disk driver supporting up to 4 disks on primary and secondary
  IDE channels. Provides LBA28 addressing for reading and writing 512-byte
  sectors. Auto-detects available disks during initialization.
  
  **DYNAMIC DISCOVERY** via PCI:
  1. Scans PCI bus for IDE controllers (class 0x01, subclass 0x01)
  2. Reads BAR0-BAR3 for channel port addresses
  3. Supports both native PCI mode and compatibility mode
  4. Falls back to legacy ports (0x1F0/0x170) if no PCI IDE found
  
  Both read and write operations use LBA mode (0xE0/0xF0 drive bits) to ensure
  correct sector addressing. The offset parameter specifies the absolute LBA
  sector number on the disk.

Device Mapping:
  0 = hda (Primary Master)
  1 = hdb (Primary Slave)
  2 = hdc (Secondary Master)
  3 = hdd (Secondary Slave)

Operations:
  read(prim_id, scnd_id, buf, offset, count)
    - Reads block(s) from IDE disk using LBA28 addressing
    - Parameters:
        prim_id: Disk ID (0-3)
        scnd_id: 0 (unused for raw disk access)
        buf: Buffer to read into (count * 512 bytes)
        offset: Starting LBA sector number (0-based, absolute on disk)
        count: Number of blocks (sectors) to read (1-256)
    - Return: Bytes read (count * 512) on success, -1 on error
    - Note: Goes through cache layer for single-block reads
    - Example: read(1, 0, buf, 100, 5) reads 5 sectors starting at LBA 100 on hdb
  
  write(prim_id, scnd_id, buf, offset, count)
    - Writes block(s) to IDE disk using LBA28 addressing
    - Parameters:
        prim_id: Disk ID (0-3)
        scnd_id: 0 (unused for raw disk access)
        buf: Data to write (count * 512 bytes)
        offset: Starting LBA sector number (0-based, absolute on disk)
        count: Number of blocks (sectors) to write (1-256)
    - Return: Bytes written (count * 512) on success, -1 on error
    - Note: Write-through cache for single-block writes
    - Example: write(1, 0, buf, 1, 1) writes 1 sector to LBA 1 on hdb
    - CRITICAL: Uses LBA mode bits (0xE0=master, 0xF0=slave) to ensure
               correct sector addressing.
  
  ioctl(prim_id, scnd_id, command)
    - NOT IMPLEMENTED (returns NULL)

Implementation Details:
  - Uses PIO (Programmed I/O) mode, not DMA
  - LBA28 addressing (max 128 GB per disk, 28-bit LBA = 2^28 sectors)
  - 512-byte sector size
  - Drive selection: 0xE0 (master) or 0xF0 (slave) with LBA bits [27:24]
  - Write command (0x30) followed by CACHE FLUSH (0xE7) per sector
  - PCI detection reads prog_if register to determine native/compatibility mode
  - Legacy ports: Primary 0x1F0/0x3F6, Secondary 0x170/0x376

Notes:
  - **DYNAMIC DISCOVERY** - PCI scan for IDE controllers
  - Offset 0 = first sector (LBA 0, typically MBR on raw disks)
  - Cache operates on single blocks to avoid thrashing
  - Multi-block I/O bypasses cache
  - All I/O goes through cache layer (cache.c) for consistency
  - Auto-detection via IDENTIFY command during init
  - Registered as /dev/hda, /dev/hdb, /dev/hdc, /dev/hdd in devfs

Devices 4-7: Partition Devices (hdaX, hdbX, hdcX, hddX)
--------------------------------------------------------------------------------
Type: Block Device
File: driver/block/part_mbr.c
Registration: mbr_init()

Description:
  MBR partition table parser and block device driver for accessing partitions.
  Parses MBR from each disk and registers partition access as block devices.
  Automatically translates partition-relative I/O to absolute disk sectors.
  
  Each disk's partitions are accessed through a single block device using
  scnd_id to select the partition number (1-4). The offset parameter specifies
  the sector offset WITHIN the selected partition (0-based).

Device Mapping:
  4 = hdaX (hda partitions 1-4)
  5 = hdbX (hdb partitions 1-4)
  6 = hdcX (hdc partitions 1-4)
  7 = hddX (hdd partitions 1-4)

Operations:
  read(prim_id, scnd_id, buf, offset, count)
    - Reads from a partition using partition-relative addressing
    - Parameters:
        prim_id: Partition device (4-7 for hda-hdd partitions)
        scnd_id: Partition number (1-4, MUST be valid partition)
        buf: Buffer to read into (count * 512 bytes)
        offset: Sector offset WITHIN partition (0-based)
        count: Number of blocks (sectors) to read
    - Return: Bytes read (count * 512) on success, -1 on error
    - Operation:
        1. Validates partition exists and scnd_id is valid (1-4)
        2. Checks bounds: offset + count <= partition_size
        3. Calculates absolute LBA: partition_start + offset
        4. Chains to underlying disk via bread(disk_id, 0, buf, absolute_lba, count)
    - Example: read(4, 1, buf, 100, 10) reads 10 sectors at offset 100
               within partition 1 of hda
  
  write(prim_id, scnd_id, buf, offset, count)
    - Writes to a partition using partition-relative addressing
    - Parameters: Same as read
    - Return: Bytes written (count * 512) on success, -1 on error
    - Operation: Same as read, but uses bwrite() to underlying disk
  
  ioctl(prim_id, scnd_id, command)
    - NOT IMPLEMENTED (returns -1)

Implementation Details:
  - Automatically translates partition-relative offsets to absolute disk LBAs
  - Comprehensive bounds checking prevents access beyond partition boundaries
  - Offset calculation: absolute_lba = partition_start + offset
  - All I/O chains through underlying disk block device (devices 0-3)
  - Cache layer automatically handles caching

Notes:
  - Only supports MBR partition tables (not GPT)
  - Supports up to 4 primary partitions per disk
  - scnd_id=0 is INVALID - partition numbers are 1-4
  - Partition types detected: FAT16, FAT32, NTFS, Minix, Linux, Linux Swap, Extended
  - Automatically validates MBR signature (0xAA55)
  - Offset 0 within partition = first sector of partition
  - Bounds checking prevents reading/writing beyond partition limits
  - Registered as /dev/hdaX, /dev/hdbX in devfs (X = partition number)

PCI (Peripheral Component Interconnect)
--------------------------------------------------------------------------------
File: driver/pci/pci.c
Header: include/driver/pci/pci.h

Description:
  PCI configuration space access and device enumeration. Supports PCI Local
  Bus Specification 2.x/3.x. Provides functions to scan for devices by class
  and subclass, and callback mechanism for device initialization.

Functions:
  - pci_init() - Initialize PCI subsystem and scan bus
  - pci_config_read_dword/word/byte() - Read configuration space
  - pci_config_write_dword() - Write configuration space
  - pci_scan_devices(class, subclass, callback) - Scan for matching devices
  - pci_enable_device(bus, slot, func, command_bits) - Enable device features

Supported Classes:
  - 0x01: Mass Storage Controller
    • 0x00: SCSI
    • 0x01: IDE Controller
    • 0x06: SATA
  - 0x02: Network Controller
  - 0x03: Display Controller
  - 0x06: Bridge Device
  - 0x07: Simple Communication Controller
    • 0x00: Serial Controller (16550-compatible)
    • 0x01: Parallel Port
    • 0x03: Modem

Notes:
  - Used by IDE driver for controller detection
  - Used by serial driver for UART detection
  - Configuration space access via ports 0xCF8/0xCFC
  - Supports BAR (Base Address Register) reading
  - Automatic device enumeration on bus 0

--------------------------------------------------------------------------------
APIC (Advanced Programmable Interrupt Controller)
--------------------------------------------------------------------------------
Files: driver/apic/lapic.c, driver/apic/ioapic.c, driver/apic/interrupt.c

Description:
  APIC subsystem providing modern interrupt handling on multi-processor
  systems. Automatically detected via ACPI MADT table. Provides unified
  interrupt management API that works with both APIC and legacy PIC.

Components:
  - Local APIC: Per-CPU interrupt controller (MSR 0x1B, base 0xFEE00000)
  - I/O APIC: External interrupt routing (base 0xFEC00000)
  - Interrupt Manager: Unified API for interrupt control

Functions:
  - interrupt_init() - Initialize interrupt controller (APIC or PIC)
  - interrupt_eoi(irq) - Send End-Of-Interrupt
  - interrupt_enable_irq(irq) - Enable interrupt line
  - interrupt_disable_irq(irq) - Disable interrupt line
  - interrupt_get_mode() - Get current mode (APIC/PIC)

Features:
  - Automatic APIC detection via ACPI MADT
  - IRQ override handling (e.g., IRQ 0 → GSI 2)
  - Automatic fallback to PIC if no APIC available
  - Mode-independent driver code (drivers use interrupt_* API)
  - Proper EOI handling for both APIC and PIC

Notes:
  - Used by PIT and keyboard drivers
  - APIC preferred over PIC when available
  - GSI (Global System Interrupt) mapping handled automatically
  - I/O APIC redirection entries configured during init

--------------------------------------------------------------------------------
ACPI (Advanced Configuration and Power Interface)
--------------------------------------------------------------------------------
Files: driver/acpi/acpi.c, driver/acpi/rsdp.c, driver/acpi/tables.c

Description:
  ACPI table parser providing system configuration information. Locates and
  validates ACPI tables including RSDP, RSDT, MADT (APIC), FADT (Power Mgmt).

Functions:
  - acpi_init() - Initialize ACPI and parse tables
  - acpi_is_available() - Check if ACPI is available
  - acpi_find_table(rsdt, signature) - Find specific table
  - acpi_shutdown() - Perform ACPI shutdown

Supported Tables:
  - RSDP: Root System Description Pointer
  - RSDT: Root System Description Table
  - MADT: Multiple APIC Description Table (for APIC init)
  - FADT: Fixed ACPI Description Table (for power management)
  - HPET: High Precision Event Timer
  - WAET: Windows ACPI Emulated Devices Table

Notes:
  - RSDP located via EBDA or BIOS ROM scan
  - Used by APIC subsystem for hardware detection
  - Used by power management for shutdown
  - All table addresses mapped via ioremap()

ABSTRACTION LAYER
================================================================================

File: driver/char/char.c, driver/block/block.c
Header: include/driver/char/char.h, include/driver/block/block.h

Description:
  Unified device driver abstraction providing consistent interface for both
  character and block devices. Supports dynamic device registration.

Public API Functions:

  cread(prim_id, scnd_id)
    - Read from character device
    - Returns: character value or 0 on error
  
  cwrite(prim_id, scnd_id, char)
    - Write to character device
    - Returns: 0 on success, -1 on error
  
  bread(prim_id, scnd_id, buf, offset, count)
    - Read from block device
    - Returns: bytes read or -1 on error
  
  bwrite(prim_id, scnd_id, buf, offset, count)
    - Write to block device
    - Returns: bytes written or -1 on error
  
  ioctl(prim_id, scnd_id, command)
    - Send control command to device
    - Returns: device-specific value or -1

Device Registration:
  - register_char_device(prim_id, ops)
  - register_block_device(prim_id, ops)

Cache Layer:
  File: driver/block/cache.c
  - LRU cache for block devices (64 entries, 512 bytes each)
  - Automatic caching on bread()
  - Write-through on bwrite()
  - Significantly improves disk I/O performance

================================================================================
SUMMARY OF DYNAMIC DISCOVERY
================================================================================

The following drivers use DYNAMIC DISCOVERY (no hardcoded addresses):

✅ Serial Ports (driver/char/serial.c)
   - PCI UART scan (class 0x07, subclass 0x00)
   - Legacy ISA port probing (COM1-COM4)
   - Validates ports before registration

✅ IDE Disks (driver/block/ide.c)
   - PCI IDE controller scan (class 0x01, subclass 0x01)
   - Reads BAR0-BAR3 for port addresses
   - Falls back to legacy ports if needed

✅ APIC (driver/apic/interrupt.c)
   - Detected via ACPI MADT table
   - Local APIC and I/O APIC addresses from MADT
   - Automatic fallback to PIC

The following drivers use HARDCODED addresses (by design):

✅ VGA (0xB8000) - Standard VGA text mode location
✅ PS/2 Keyboard (0x60) - Standard PS/2 port
✅ PIT Timer (0x40-0x43) - Standard 8253/8254 ports
✅ RTC/CMOS (0x70-0x71) - Standard CMOS ports

These hardcoded addresses are PC/AT legacy standards and do not have
discovery mechanisms. They are correct and should remain hardcoded.
